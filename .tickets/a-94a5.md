# Implement stage page server load function

Create +page.server.ts to load stage data, user submissions, and conditionally other submissions.

## Design

File: src/routes/b/[id]/s/[id]/+page.server.ts

Load:
- Stage with battle relation (db.query.stage.findFirst with battle relation)
- User's submissions for this stage
- Other submissions only if phase is voting or closed
- Compute canSubmit boolean from phase, deadline, existing submission count

Checks:
- Auth required (redirect to /signin if not logged in)
- User must be player or creator (error 403 if not)

Return: { stage, battle, userSubmissions, otherSubmissions, canSubmit, maxSubmissions, user }

## Acceptance Criteria

- Page loads stage and battle data
- User's submissions returned
- Other submissions only returned in voting/closed phases
- canSubmit correctly computed
- Non-participants get 403
